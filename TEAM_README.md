## Scenario
Scenario 1: Code Modernization

## Client & Problem Statement

**Client:** QueenFiona Real Estate
**System:** Property Listings Portal
**Built:** 2016 in AngularJS 1.x

QueenFiona's property listings portal has become an active liability. The system was built 8 years ago by a vendor who is no longer engaged, and no one currently on staff can safely modify it. The codebase has no tests, no documentation, and tribal knowledge left with the previous team.

AngularJS reached end of life in December 2021. The portal has received no security patches in over three years. Third-party API keys â including the map provider and listing data feed â are hardcoded in client-side JavaScript, visible to anyone who opens browser DevTools. QueenFiona is one disclosure away from a regulatory incident.

Every feature request requires weeks of investigation before a line of code is written, because no one understands the system's dependencies or side effects. The last two attempts to add functionality introduced regressions in unrelated parts of the portal. As a result, the product roadmap has been frozen for 18 months.

The business cannot grow on this foundation. QueenFiona needs to launch property alerts, agent dashboards, and a mobile experience within the next two quarters. None of that is possible without modernizing what exists first.

**The mandate:** assess the current system, establish a safe base for change, and begin extracting the core capabilities into a maintainable architecture â without taking the portal offline.
